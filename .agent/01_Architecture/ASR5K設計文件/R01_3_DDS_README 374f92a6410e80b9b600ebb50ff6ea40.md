# R01_3_DDS_README

# DDS 控制系統與優化分析報告

## 📋 功能說明

這是一個為 TI C2000 F28388D 平台量身打造的高效能 DDS (Direct Digital Synthesis) 控制系統。其核心設計目標是實現 **100KHz 高頻 ISR 下的極低 CPU 佔用率**，同時保持完整的狀態機控制功能。

### 主要功能特性

- **高精度波形生成**: 支援 100KHz 採樣率，使用 4096 點高解析度正弦表（儲存於專用 RAMGS4 記憶體區段）。
- **智慧型狀態機**: 完整的生命週期管理 (Idle -> Init Table -> Stopped -> Delay On -> Running -> Amp Ramp Down -> Delay Off -> Phase Off -> Stopped)。
- **精確時序控制**: 支援可中斷的 Delay ON/OFF (0~60秒) 與 Phase ON/OFF (-1 立即執行或 0-360 度鎖定)。
- **平滑過渡 (Ramp)**: 針對振幅、偏移、頻率提供獨立的斜坡控制 (Ramp)，支援動態參數更新與目標追蹤。
- **DC 模式 (0 Hz)**: 支援設定頻率為 0 Hz，實現純 DC 輸出（自動鎖定有效振幅為 0，輸出特定 Offset），並支援與 AC 模式間的平滑切換。
- **智能停止緩降**: 停止時若啟用振幅斜坡，系統會自動進入 AMP_RAMP_DOWN 狀態，將振幅緩降至 0 後才進入 Phase Off，確保輸出不會有突波。
- **極致效能優化**: 針對 C28x 架構進行了指令級優化，包含 Static Inline、除法消除、Ramp 旁路與狀態列舉重排（詳見後文分析）。

## 🔄 狀態流程 (State Machine)

狀態機設計為可搶佔式 (Preemptive)，允許在 Delay 或 Phase 階段隨時響應 Start/Stop 命令。

1. **IDLE**: 系統初始狀態（未使用）。
2. **INIT_TABLE**: 系統啟動，進行非阻塞式正弦表建構（每次呼叫 `runDDS()` 建構 1 點，共 4096 點）。
3. **STOPPED**: 建表完成，輸出維持 DC Offset，等待啟動指令。
    - *Event: DDS_Start()* -> 進入 DELAY_ON
4. **DELAY_ON**: 執行啟動延遲計數。輸出維持 DC Offset，相位歸零。
    - *Event: 計數完成* -> 根據 Phase ON 設定進入 RUNNING
        - 若 `s16PhaseOnDegrees == -1`: 從 0 度開始
        - 若 `s16PhaseOnDegrees == 0~360`: 直接設定相位暫存器至目標角度
    - *Event: DDS_Stop()* -> 立即回到 STOPPED（可中斷）
5. **RUNNING**: 正常波形輸出。執行 Ramp 邏輯與相位累加。
    - *Event: DDS_Stop() with AmpRamp Enabled* -> 進入 AMP_RAMP_DOWN
    - *Event: DDS_Stop() without AmpRamp* -> 進入 DELAY_OFF
6. **AMP_RAMP_DOWN**: 振幅緩降狀態。自動將振幅斜坡目標設為 0，緩降至 0 後進入 PHASE_OFF。
    - *Event: Amp Ramp Done* -> 進入 PHASE_OFF
    - *Event: DDS_Start()* -> 立即回到 RUNNING（可中斷）
7. **DELAY_OFF**: 執行停止延遲計數。波形持續輸出，相位持續累加。
    - *Event: 計數完成* -> 根據 Phase OFF 設定進入 PHASE_OFF 或 STOPPED
        - 若 `s16PhaseOffDegrees == -1`: 立即進入 STOPPED
        - 若 `s16PhaseOffDegrees == 0~360`: 進入 PHASE_OFF
    - *Event: DDS_Start()* -> 立即回到 RUNNING（可中斷）
8. **PHASE_OFF**: 尋找結束相位。波形持續輸出，使用鎖定的相位增量（防止 Ramp 干擾）。
    - *Event: 相位穿越目標角度* -> 回到 STOPPED，相位歸零
9. **ERROR**: 錯誤狀態，輸出維持 DC Offset。

## 🏗️ 架構設計與優化分析 (Architecture & Optimization)

本系統針對 C28x DSP 進行了四項關鍵優化，使得 ISR 執行時間從優化前的 **>5us** 大幅降低至 **<2.5us**。

### 1. Inlining (函數內聯)

- **設計理念**: 將 ISR 關鍵路徑函數 `stepDDS` 和 `getDDS` 定義於 `dds_core.h` 並宣告為 `static inline`。
- **實作方式**: 所有 API 函數（`dds_api.h`）與 Ramp 運算（`dds_timectrl.h`）均採用 `static inline` 設計。
- **效益**: 編譯器直接將代碼展開至 ISR 中，消除函數呼叫的堆疊操作（`ASP`、`NASP`）和分支指令（`LCR`），節省約 5-10 cycles。

### 2. Division Removal (除法消除)

- **問題分析**: 相位增量計算 `(Freq * 2^32) / (SampleRate * 100)` 需要 64-bit 除法，C28x 執行極慢（>50 cycles）。
- **優化策略**: 在 `DDS_Init()` 中預先計算 `f32PhaseStepFactor = 2^32 / (SampleRate * 100)`。
- **執行時運算**: 僅需單次浮點乘法 `u32PhaseIncrement = (uint32_t)(Freq_x100 * f32PhaseStepFactor)`。
- **效益**: 將除法替換為單週期 FPU 乘法（`MPYF32`），大幅降低 ISR 負載。

### 3. Target Tracking Optimization (目標追蹤優化)

- **設計理念**: 在 `ST_DDS` 結構中新增 `u16AmpRampTarget`、`u16OffsetRampTarget`、`u32FreqRampTarget` 三個追蹤變數。
- **實作方式**: `stepDDS()` 中僅在目標值改變時才更新 Ramp 目標（`f32Target`），避免每次 ISR 都執行浮點轉換。
- **效益**: 在穩態下（參數未變動），跳過不必要的浮點運算，降低 CPU 負載。

### 4. Enum Reordering (狀態列舉重排)

- **設計理念**: 將需要輸出波形的狀態（`RUNNING`, `DELAY_OFF`, `PHASE_OFF`, `AMP_RAMP_DOWN`）集中在 Enum 後段。
- **實作方式**: 定義 `DDS_STATE_OUTPUT_START` 作為分界標記，輸出狀態值均 >= 此標記。
- **效益**: `getDDS()` 中的狀態檢查可簡化為單一範圍比較（未在當前實作中使用，但架構已預留）。

### 5. Phase Lock for Phase OFF (相位鎖定機制)

- **問題分析**: 在 PHASE_OFF 狀態下，若 Frequency Ramp 仍在運作，會導致相位增量持續變化，無法準確命中目標角度。
- **優化策略**: 進入 PHASE_OFF 前，將當前的 `u32PhaseIncrement` 鎖定至 `u32PhaseOffIncrement`。
- **實作方式**: PHASE_OFF 狀態使用 `u32PhaseOffIncrement` 進行相位累加，而非動態的 `u32PhaseIncrement`。
- **效益**: 確保 Phase OFF 角度控制的精確性，不受 Ramp 干擾。

## 🚀 快速開始 (Quick Start)

### 1. 初始化 (Setup)

在 `main()` 函數中進行初始化，並在低頻 Task 中建立正弦表。

```c
#include"dds/dds_api.h"

void main(void) {
    // 1. 初始化: 100KHz 採樣, 初始 50.00Hz, 0.5 PU 振幅, 0.5 PU 偏移
    DDS_Init(100000UL, 5000UL, 32767U, 32768U);

    // 2. 等待建表 (同步方式示例)
    while(!DDS_IsInitComplete()) {
        DDS_Poll();  // 每次呼叫建構 1 點，共需 4096 次
    }

    // 3. 啟動 ISR
    // ...
}
```

### 2. ISR 整合 (Integration)

在 100KHz 中斷服務程式中調用核心函數。

```c
interrupt void ISR_100KHz(void) {
    // [關鍵順序] 1. 先執行狀態步進 (包含 Ramp 更新與相位累加)
    DDS_Step();

    // [關鍵順序] 2. 獲取計算後的樣本 (查表與振幅縮放)
    uint16_t u16Out = DDS_GetSample();

    // 3. 寫入 DAC (移除低 4 位，適配 12-bit DAC)
    DAC_setShadowValue(DACA_BASE, u16Out >> 4);
}
```

### 3. 運行時控制 (Runtime Control)

在應用層 Task (如 `timetask.c`) 中控制 DDS。

```c
void Task_10ms(void) {
    // 持續呼叫以完成正弦表建構 (建表期間)
    DDS_Poll();

    // 改變頻率為 60.00 Hz (自動應用除法優化)
    DDS_SetFrequency(6000UL);

    // 設定 1.5秒 的振幅 Ramp 到 1.0 PU
    DDS_SetAmplitude(65535U);            // 設定目標振幅
    DDS_SetAmplitudeRamp(1.5f, 1.5f, 1); // 啟動 Ramp (Up=1.5s, Down=1.5s)

    // 設定 Delay 與 Phase 控制
    DDS_SetDelayOn(0.5f);   // 啟動延遲 0.5 秒
    DDS_SetDelayOff(0.2f);  // 停止延遲 0.2 秒
    DDS_SetPhaseOn(0.0f);   // 從 0 度開始
    DDS_SetPhaseOff(180.0f); // 在 180 度停止

    if (StartButton) DDS_Start();
    if (StopButton)  DDS_Stop();
}
```

## 📝 函數使用說明 (API Reference)

所有 API 均定義在 `dds_api.h`，宣告為 `static inline` 以確保效能。

### 核心控制

| 函數 | 說明 | 備註 |
| --- | --- | --- |
| `DDS_Init(...)` | 初始化 DDS 結構與參數 | 需傳入 SampleRate, Freq, Amp, Offset |
| `DDS_Start()` | 發送啟動請求 | 狀態機轉為 DELAY_ON |
| `DDS_Stop()` | 發送停止請求 | 狀態機轉為 DELAY_OFF |
| `DDS_Step()` | **[ISR專用]** 執行狀態機與 Ramp | 必須在 GetSample 前呼叫 |
| `DDS_GetSample()` | **[ISR專用]** 計算並回傳樣本 | 高效查表實作 |
| `DDS_Poll()` | **[Task專用]** 建構正弦表 | 每次呼叫建 1 點 |

### 參數設定

| 函數 | 參數格式 | 說明 |
| --- | --- | --- |
| `DDS_SetFrequency(u32)` | Freq * 100 | 設定目標頻率。0: 進入 DC 模式 (振幅無效)100~100000: AC 模式 (1.00 - 1000.00 Hz) |
| `DDS_SetAmplitude(u16)` | 0 - 65535 | 設定目標振幅 (0.0 - 1.0 PU) |
| `DDS_SetOffset(u16)` | 0 - 65535 | 設定目標偏移 (32768 為中點) |

### Ramp 控制

所有 Ramp 設定函數格式均為 `(float UpTime, float DownTime, bool Enable)`。
* `DDS_SetAmplitudeRamp(...)`
* `DDS_SetFrequencyRamp(...)`
* `DDS_SetOffsetRamp(...)`

### 時序與相位

- `DDS_SetDelayOn(float sec)` / `DDS_SetDelayOff(float sec)`
- `DDS_SetPhaseOn(float deg)` / `DDS_SetPhaseOff(float deg)`

### 狀態查詢

- `DDS_GetState()` - 取得當前狀態
- `DDS_IsRunning()` - 檢查是否在 RUNNING 狀態
- `DDS_IsInitComplete()` - 檢查初始化是否完成
- `DDS_GetFrequency()` / `DDS_GetAmplitude()` / `DDS_GetOffset()` - 取得當前參數
- `DDS_IsRampComplete(u16Index)` - 檢查特定 Ramp 是否完成（0=Amp, 1=Offset, 2=Freq）
