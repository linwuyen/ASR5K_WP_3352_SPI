# [音檔說明]DDS控制與保護系統架構整理

[說明]本篇針對音檔內容，詳細梳理DDS訊號輸出、PWM控制、系統校準、補償機制，以及硬體安全保護與錯誤備份等核心開發流程，提供條列式架構說明。

## 核心開發與功能實現流程

- **基礎功能建置：**
    - 必須先完成將 DDS 訊號成功輸出（打出來）給其他硬體端。
    - 在輸出前，需優先完成所有控制 DDS 的相關命令與控制機制的開發，以確保後續能夠有效操控。
- **命令通道打通：**
    - 將基本的控制命令與主控端（如 3352）進行對接與打通。
    - 打通後，即可延用 3352 系統既有的命令集來直接控制 DDS。
- **周邊開關與 PWM 輸出配置：**
    - 訊號輸出目前僅到達 DAC 端，若要成功發射出去，需開啟 M0晶片內部的相關硬體開關（如 Range 切換）。 AD_SDI_EN ON 才會進LTC
    - 必須開啟 PWM 功能，並正確配置其比較參數（Compare Parameters），PWM 訊號才能真正成功輸出。 M0 的 CLK0 CLK180 ON，DDS 經過OPA 算出ERR

[**設定校準（Setting Calibration）**](%E8%A8%AD%E5%AE%9A%E6%A0%A1%E6%BA%96%EF%BC%88Setting%20Calibration%EF%BC%89%20374f92a6410e803fa314f0290e769d11.md)

[剖析硬體非線性誤差來源與校準補償策略說明](%E5%89%96%E6%9E%90%E7%A1%AC%E9%AB%94%E9%9D%9E%E7%B7%9A%E6%80%A7%E8%AA%A4%E5%B7%AE%E4%BE%86%E6%BA%90%E8%88%87%E6%A0%A1%E6%BA%96%E8%A3%9C%E5%84%9F%E7%AD%96%E7%95%A5%E8%AA%AA%E6%98%8E%20373f92a6410e8005a802f62b349b9b3a.md)

- **設定與校準：**
    - 訊號打出去後，為確保精準度，需進行系統校準，主要包含**電壓校準**與**頻率校準**。
    
    拿標準儀器、固定負載 建立：Gain、Offset 、Frequency
    
    電壓校準 analog chain 每一段都會有誤差，線性校正
    
    - 校正作法
        
        電源供應器的 Calibration 本質上是在做：
        
        # 「建立數位世界 ↔ 真實物理世界的映射」
        
        因為：
        
        ```
        DAC code ≠ 真正輸出電壓
        DDS frequency word ≠ 真正輸出頻率
        ADC reading ≠ 真實量測值
        ```
        
        所以：
        
        # Calibration 的核心：
        
        就是建立：
        
        ```
        digital_value → real_world_value
        ```
        
        的修正模型。
        
        ---
        
        # 一、真正的 Calibration 架構
        
        工業級通常分：
        
        ```
        Factory Calibration
        Runtime Compensation
        Field Re-Calibration
        ```
        
        ---
        
        # 1. Factory Calibration（最重要）
        
        出廠時做。
        
        通常：
        
        - 室溫
        - 標準儀器
        - 固定負載
        
        目的：
        
        建立：
        
        - Gain
        - Offset
        - Frequency correction
        
        ---
        
        # 2. Runtime Compensation
        
        運行中：
        
        - 溫漂
        - 線損
        - amplifier drift
        
        動態修正。
        
        ---
        
        # 3. Field Re-Calibration
        
        維修模式。
        
        例如：
        
        SCPI command：
        
        ```
        CAL:VOLT 10V
        ```
        
        重新校正。
        
        ---
        
        # 二、電壓 Calibration 怎麼做
        
        真正核心：
        
        # 校正的是整條 analog chain
        
        例如：
        
        ```
        DDS
        → DAC
        → OPAMP
        → Power stage
        → Output terminal
        ```
        
        因為每段都有誤差。
        
        ---
        
        # 最常見校正：
        
        ## 1. Offset Calibration
        
        修正：
        
        ```
        輸出應該 0V
        實際變 0.023V
        ```
        
        ---
        
        ## 作法
        
        輸出：
        
        ```
        DAC = 0
        ```
        
        然後：
        
        用高精度 DMM 量。
        
        假設：
        
        ```
        Measured = 0.023V
        ```
        
        則：
        
        ```
        offset = -0.023
        ```
        
        存進 Flash。
        
        ---
        
        # Runtime：
        
        ```c
        Vout_corrected = Vout_target + offset;
        ```
        
        ---
        
        # 2. Gain Calibration
        
        修正比例誤差。
        
        例如：
        
        ```
        Target = 10.000V
        Measured = 9.82V
        ```
        
        表示：
        
        gain 有誤差。
        
        ---
        
        # 作法
        
        輸出幾個 calibration point：
        
        例如：
        
        ```
        1V
        5V
        10V
        ```
        
        用 DMM 量測。
        
        ---
        
        # 建立：
        
        ## 線性模型
        
        V_{real}=aV_{command}+b
        
        其中：
        
        - a = gain correction
        - b = offset correction
        
        ---
        
        # 實際 runtime：
        
        ```c
        dac_command =
        (target_voltage - offset)
        /
        gain;
        ```
        
        ---
        
        # 三、為什麼不能只校正一點
        
        因為：
        
        analog system 不一定線性。
        
        例如：
        
        - op amp rail limitation
        - DAC INL
        - amplifier saturation
        - power stage nonlinearity
        
        所以工業級：
        
        通常：
        
        # Multi-point Calibration
        
        例如：
        
        ```
        0V
        2V
        5V
        8V
        10V
        ```
        
        建立：
        
        # LUT（lookup table）
        
        ---
        
        # Runtime：
        
        ```
        target_voltage
        → interpolation
        → corrected DAC code
        ```
        
        這比單純 gain/offset 準。
        
        ---
        
        # 四、頻率 Calibration 怎麼做
        
        DDS 真正誤差來源：
        
        ---
        
        ## 1. Reference Clock Error
        
        例如：
        
        ```
        25MHz crystal
        實際 = 24.9991MHz
        ```
        
        DDS 全部頻率都偏。
        
        ---
        
        ## 2. PLL 誤差
        
        ---
        
        ## 3. 溫漂
        
        ---
        
        # DDS 頻率公式
        
        f_{out}=\frac{FTW\times f_{clk}}{2^N}
        
        ---
        
        # 真正 Calibration：
        
        不是修 FTW。
        
        而是：
        
        # 修正 f_clk
        
        例如：
        
        量到：
        
        ```
        實際輸出：
        999.82Hz
        ```
        
        原本目標：
        
        ```
        1000Hz
        ```
        
        則：
        
        ```
        clock_correction =
        1000 / 999.82
        ```
        
        ---
        
        # Runtime：
        
        ```c
        effective_clk =
        nominal_clk * correction;
        ```
        
        再重新算 FTW。
        
        ---
        
        # 五、校正流程（真正工廠做法）
        
        真正流程通常：
        
        ---
        
        # Step 1
        
        設備進 calibration mode
        
        ```
        NORMAL MODE 禁止
        ```
        
        避免誤操作。
        
        ---
        
        # Step 2
        
        接：
        
        - Precision DMM
        - Frequency Counter
        - Electronic Load
        
        ---
        
        # Step 3
        
        輸出 calibration points
        
        例如：
        
        ```
        1V
        5V
        10V
        ```
        
        與：
        
        ```
        100Hz
        1kHz
        100kHz
        ```
        
        ---
        
        # Step 4
        
        讀取儀器值
        
        建立：
        
        - gain
        - offset
        - LUT
        
        ---
        
        # Step 5
        
        寫入 Flash
        
        例如：
        
        ```c
        typedef struct
        {
            float volt_gain;
            float volt_offset;
        
            float freq_correction;
        
            float temp_coeff;
        } CAL_DATA;
        ```
        
        ---
        
        # 六、校正後怎麼驗證（超重要）
        
        很多人只會：
        
        > 「校正成功」
        > 
        
        但：
        
        # Calibration ≠ Validation
        
        真正重要的是：
        
        # 驗證 residual error。
        
        ---
        
        # 驗證核心：
        
        ```
        commanded_value
        vs
        measured_value
        ```
        
        差多少。
        
        ---
        
        # 1. 電壓驗證
        
        例如：
        
        測：
        
        ```
        1V
        3V
        5V
        7V
        10V
        ```
        
        ---
        
        # 建立誤差表：
        
        | Target | Measured | Error |
        | --- | --- | --- |
        | 1V | 1.001V | 1mV |
        | 5V | 4.998V | -2mV |
        
        ---
        
        # 驗證：
        
        ## 最大誤差
        
        Error%=\frac{V_{measured}-V_{target}}{V_{target}}\times100%
        
        ---
        
        # 常見規格：
        
        ```
        <0.1%
        <0.05%
        <10mV
        ```
        
        ---
        
        # 七、頻率驗證
        
        用：
        
        - Frequency Counter
        - Oscilloscope
        
        測：
        
        ```
        100Hz
        1kHz
        10kHz
        1MHz
        ```
        
        ---
        
        # 驗證：
        
        ## ppm error
        
        ppm=\frac{f_{measured}-f_{target}}{f_{target}}\times10^6
        
        ---
        
        # 八、真正工業級還會做
        
        # Temperature Sweep Validation
        
        因為：
        
        室溫準沒意義。
        
        真正問題：
        
        ```
        25°C OK
        70°C 飄掉
        ```
        
        ---
        
        # 所以真正 validation：
        
        會進：
        
        - thermal chamber
        
        測：
        
        ```
        -20°C
        25°C
        70°C
        ```
        
        看：
        
        - voltage drift
        - frequency drift
        
        ---
        
        # 九、最容易被忽略的東西
        
        # Calibration Traceability
        
        真正工業系統：
        
        必須記錄：
        
        ```
        校正時間
        校正人員
        儀器序號
        儀器精度
        環境溫度
        FW version
        ```
        
        否則：
        
        客訴時根本追不到。
        
        ---
        
        # 十、真正核心理解（很重要）
        
        Calibration 本質不是：
        
        > 「讓數字變準」
        > 
        
        而是：
        
        # 「建立可預測 deterministic system」
        
        因為 control system 最怕：
        
        ```
        command ≠ reality
        ```
        
        一旦 mapping 不穩：
        
        - PI loop
        - compensation
        - protection threshold
        
        全部都會錯。
        
    - range switching 電壓、電流高低檔
        
        因為真正的電源供應器：
        
        # 「不是一個線性的世界」
        
        尤其：
        
        - DAC
        - ADC
        - OPAMP
        - Current Sense
        - PGA
        - Power Stage
        
        都存在：
        
        ```
        dynamic range limitation
        ```
        
        所以：
        
        # 同一組硬體參數
        
        不可能同時兼顧：
        
        - 小訊號精度
        - 大訊號範圍
        
        因此才需要：
        
        # 高低檔（Range）
        
        ---
        
        # 一、什麼叫高低檔
        
        例如：
        
        一台 power supply：
        
        ---
        
        # Voltage Range
        
        ```
        0~10V   → Low Range
        0~100V  → High Range
        ```
        
        ---
        
        # Current Range
        
        ```
        0~100mA → Low Current Range
        0~10A   → High Current Range
        ```
        
        ---
        
        # 本質：
        
        # 「切換不同 analog gain」
        
        不是單純軟體變數。
        
        ---
        
        # 二、為什麼一定需要 Range
        
        因為：
        
        # ADC / DAC 解析度有限
        
        例如：
        
        12-bit ADC：
        
        Resolution=\frac{Full\ Scale}{2^{12}}
        
        ---
        
        # 假設：
        
        ## 100V Full Scale
        
        則：
        
        ```
        LSB ≈ 24mV
        ```
        
        這代表：
        
        你根本不可能精準量：
        
        ```
        1mV
        ```
        
        ---
        
        # 但如果切：
        
        ## 10V Range
        
        則：
        
        ```
        LSB ≈ 2.4mV
        ```
        
        解析度瞬間提升。
        
        ---
        
        # 三、Current Range 更明顯
        
        這是最常見。
        
        因為：
        
        # 大電流與小電流差太多。
        
        例如：
        
        ---
        
        # High Current Range
        
        ```
        0~10A
        shunt = 10mΩ
        ```
        
        10A 時：
        
        genui{"math_block_widget_always_prefetch_v2":{"content":"V=IR"} }
        
        得到：
        
        ```
        100mV
        ```
        
        OK。
        
        ---
        
        # 但：
        
        1mA 時：
        
        ```
        10uV
        ```
        
        ADC 根本看不到。
        
        ---
        
        # 所以：
        
        需要：
        
        # Low Current Range
        
        例如：
        
        ```
        shunt = 1Ω
        ```
        
        這樣：
        
        1mA：
        
        ```
        1mV
        ```
        
        才能量。
        
        ---
        
        # 四、真正 Range 是怎麼切
        
        通常：
        
        ---
        
        # 1. Relay
        
        最準。
        
        切：
        
        - feedback resistor
        - shunt
        - gain path
        
        缺點：
        
        - 慢
        - 有壽命
        
        ---
        
        # 2. Analog Switch
        
        例如：
        
        - ADG series
        - TMUX
        
        快。
        
        但：
        
        - leakage
        - Ron
        - charge injection
        
        會影響精度。
        
        ---
        
        # 3. PGA（Programmable Gain Amplifier）
        
        直接切：
        
        ```
        x1
        x10
        x100
        ```
        
        ---
        
        # 五、為什麼 Calibration 要分高低檔
        
        因為：
        
        # 每個 Range 的誤差都不同。
        
        例如：
        
        ---
        
        # Low Range
        
        ```
        Gain = 9.982
        Offset = 1mV
        ```
        
        ---
        
        # High Range
        
        ```
        Gain = 100.24
        Offset = 18mV
        ```
        
        ---
        
        # 所以：
        
        真正 calibration data：
        
        不是：
        
        ```c
        float gain;
        float offset;
        ```
        
        而是：
        
        ```c
        typedef struct
        {
            float gain;
            float offset;
        
        } RANGE_CAL;
        ```
        
        ---
        
        # 然後：
        
        ```c
        typedef struct
        {
            RANGE_CAL volt_low;
            RANGE_CAL volt_high;
        
            RANGE_CAL curr_low;
            RANGE_CAL curr_high;
        
        } CAL_DATA;
        ```
        
        ---
        
        # 六、真正 Runtime 怎麼用
        
        例如：
        
        ---
        
        # 使用者設定：
        
        ```
        20mA
        ```
        
        系統：
        
        ```
        auto select low current range
        ```
        
        ---
        
        # 使用者設定：
        
        ```
        5A
        ```
        
        系統：
        
        ```
        switch high current range
        ```
        
        ---
        
        # 七、真正難的是：
        
        # Range Switching Transient
        
        這超容易炸。
        
        因為：
        
        切換瞬間：
        
        - gain 改變
        - feedback 改變
        - loop gain 改變
        
        可能：
        
        # control loop 突然失穩。
        
        ---
        
        # 所以真正工業系統：
        
        切換時：
        
        ```
        disable PWM
        wait settle
        switch range
        recalibrate ADC
        restart loop
        ```
        
        ---
        
        # 八、真正高階系統
        
        甚至：
        
        # 每個 Range 有獨立控制器參數
        
        例如：
        
        ---
        
        # Low Current Range
        
        PI：
        
        ```
        Kp = 0.1
        Ki = 0.01
        ```
        
        ---
        
        # High Current Range
        
        PI：
        
        ```
        Kp = 0.4
        Ki = 0.1
        ```
        
        因為：
        
        plant 不一樣。
        
        ---
        
        # 九、你現在開始碰到：
        
        # Instrument-grade power supply architecture
        
        這已經不是：
        
        「PWM 控制」。
        
        而是：
        
        # Precision mixed-signal system。
        
        ---
        
        # 十、真正第一性原理
        
        高低檔存在的本質：
        
        不是：
        
        > 「功能比較多」
        > 
        
        而是：
        
        # 在有限 ADC/DAC resolution 下
        
        同時滿足：
        
        ```
        大範圍
        +
        高精度
        ```
        
        這是：
        
        所有量測儀器：
        
        - DMM
        - Oscilloscope
        - SMU
        - Power Supply
        
        都一定存在 range switching 的根本原因。
        
- **效能驗證：**
    - 校準完成後，需進一步驗證與量測輸出訊號的電壓與頻率是否達到預期精準度。
- **參數補償機制：**
    - 當後續導入電流控制時，需考慮線路損耗（線損）問題。
    - 系統需規劃並寫入**電流補償**與**溫度補償**等演算法或修正機制。

## 硬體保護與錯誤處理機制

- **保護功能優先開啟：**
    - 在尚未建置保護功能前，切勿直接進行規格驗規，以免發生硬體損壞危險。
    - Protection > Control
        
        
- **硬體錯誤偵測項目：**
    - 系統須透過 M0 晶片持續監測硬體線路，包含以下異常狀態：
        - **OVP**（過電壓保護）
        - **OCP**（過電流保護）
        - **OTP**（過溫保護）
        - **PFC** 狀態異常
        - **風扇運轉**異常
    
    M0 觸發 OVP 把這BIT 傳給 C28 ，3352 background ****POLLING C28 的 BIT
    
- **異常連鎖反應（Interlock）：**
    - 當上述任一錯誤觸發時，系統必須**立即關閉 PWM 輸出**。
    - 同時向 3352 主控端發出通報，說明當前發生的錯誤類型。
- **錯誤日誌儲存與備份：**
    - 異常發生時，系統需自行將錯誤代碼與相關數據紀錄下來。
    - **儲存策略：** 先暫存於運行速度較快的 RAM 中，待系統閒置（有閒時）時，再將資料寫入 Flash 進行永久備份。
