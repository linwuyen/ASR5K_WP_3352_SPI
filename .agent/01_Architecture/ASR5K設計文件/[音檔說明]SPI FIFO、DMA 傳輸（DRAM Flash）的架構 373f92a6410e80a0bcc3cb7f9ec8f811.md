# [音檔說明]SPI FIFO、DMA 傳輸（DRAM/Flash）的架構

這段音訊主要探討了在**高速連續資料傳輸**中，如何透過 **SPI FIFO、DMA 傳輸與多級緩衝區（DRAM/Flash）**的架構優化，來避免資料發生遺失（Loss）的硬體設計與計算邏輯。
以下為您詳細整理音檔內容與核心量化計算：

## 類別與目的

- **[類別]** 資料傳輸與嵌入式系統架構
- **[目的]** 高速連續資料傳輸下，避免 FIFO 溢位與緩衝區優化計算說明

## 一、 連續資料傳輸與 FIFO 溢位風險

兩位工程師針對主控端（如 3352）連續傳送資料時的硬體瓶頸進行了分析：

- **連續傳輸背景：**
主控端在傳送一個 4096 筆（點）的資料包時，採用的是**連續發送**（Continuous Transmission）。資料點與資料點之間的**間隔非常短**，通常僅有

$$
 1 \sim 2\ \mu\text{s}，
$$

        甚至可能更短至數百個奈秒（\text{ns}）。

- **SPI FIFO 的硬體限制：**
接收端的 SPI FIFO 硬體容量**僅有 16 層（Level）**，意即在未被取走的情況下，它最多只能連續接收 16 筆資料。
- **資料遺失（Loss）原因：**
當 4096 筆資料以極高速度連續塞入時，如果沒有一個更快速的機制將資料從 16 層的 FIFO 中搬走，FIFO 很快就會全滿。後續源源不絕進來的資料便會因為溢位而**全部遺失（Loss）**。

## 二、 DMA 與中斷（{R}_X）的連鎖反應解法

為了解決 16 層 FIFO 太小、容易溢位的問題，系統必須導入 DMA（直接記憶體存取）架構來擴展緩衝空間：
【主控端連續發送】
│
▼ (間隔 1~2 μs)
【SPI FIFO (僅 16 層)】 ──► 觸發 RX 中斷 ──► 【DMA 搬運】 ──► 【擴展緩衝空間 (DRAM)】

- **觸發機制：** 當資料進入並填入 FIFO 時，必須利用 SPI 的接收中斷（{R}_X 中斷）來自動觸發 DMA 傳輸。
- **DMA 搬運任務：** DMA 被觸發後，負責以極快速度將資料從 FIFO 逐一搬運至更大的外部記憶體中（例如 SDRAM 或 Global Range），相當於將硬體上限僅 16 層的 FIFO **在軟體與記憶體層面「擴展」開來**。
- **後級儲存與輪詢（While Loop）：**
搬運到 SDRAM 的資料只是暫存，最終應用（如存檔）需要將其寫入 **Flash** 儲存。此時必須在主程式的 while(1) 迴圈中，持續檢查 Global RAM 是否快滿了，並即時將資料搬往 Flash。這需要確保 while 迴圈的**執行時間（Time Budget）足夠**，才能將資料順利搬完。

### 4. 緩衝區（Global Range）容量與安全的取捨

為了給 while 迴圈爭取更多處理時間，Global Range 開得越大，系統就越安全，但會犧牲 RAM 空間。以下是兩種極端與優化方案的評估：

### 方案 A：極端安全法（開足 4096 點空間）

- **作法：** 直接配合主控端的封包，開一個完整包含 4096 點資料的 Global RAM 來接收。
- **缺點：** 雖然不需要頻繁擔心時間不夠，但這會**極度消耗珍貴的 RAM（DRAM）空間**。在實際產品設計中，通常不需要開到這麼大。
