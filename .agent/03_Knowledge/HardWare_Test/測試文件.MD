﻿# SPI Master / Slave 功能測試文件

本文件依照目前 `Emu_3352_SPI` 程式碼編寫，用於 CCS 逐項驗證 SPIA Master 與 SPIB Slave。

測試原則：

- 正常測試只操作 `spiA_master.stTest`。
- 每次只執行一個測試。
- Counter 使用測試前後差值判定，不假設上電後一定為 0。
- `SPI_TEST_STATUS_PASSED` 只代表 Master 測試流程通過；仍須核對 Slave 狀態與資料。
- 發生 Fault 後先記錄診斷內容，再 Reset CPU 重新測試。

---

## 1. 目前可測功能

| Command | 數值 | 功能 |
|---|---:|---|
| `SPI_MASTER_TEST_CMD_WRITE` | 1 | Legacy 2-word register write |
| `SPI_MASTER_TEST_CMD_READ` | 2 | Legacy 2-word register read |
| `SPI_MASTER_TEST_CMD_SEQ_WRITE_16` | 3 | 16-word block write |
| `SPI_MASTER_TEST_CMD_WAVE_4095` | 4 | 4095-point ramp transfer |
| `SPI_MASTER_TEST_CMD_SINE_4095` | 5 | 4095-point sine transfer |
| `SPI_MASTER_TEST_CMD_REG_FRAME_1000` | 6 | 1000-frame DMA stress test |
| `SPI_MASTER_TEST_CMD_PACKET_WRITE` | 7 | Header/Command/Length/Payload/Checksum packet write |

目前兩種協定共存：

1. Legacy 2-word register frame：`Address/Command + Data`
2. Packet mode：`0xA55A + Command + Length + Payload + Checksum`

一般 register command 會共用 Slave register handler；Packet block command 則直接使用
同一套 block receiver。

## 4. CCS Expressions 設定

### 4.1 正常測試必看

加入：

```txt
spiA_master.stTest
spiA_master.stDiag
spiB_slave.stDiag
gSpibRxDmaDoneCount
gSpibRxParseOkCount
gSpibRxParseFailCount
gSpibRxDmaRestartCount
gSpibRxErrorFlags
```

Counter 判定注意：

- `gSpibRxDmaDoneCount`：DMA 實際完成的 2-word frame 數。
- `gSpibRxParseOkCount`：應與 `gSpibRxDmaDoneCount` 同步增加。
- `gSpibRxDmaRestartCount`：除每筆 frame 重啟外，2 ms silence recovery 也會再增加一次。
- Block 測試完成前會額外執行 Status read；每次 Status read 會增加 2 筆 DMA frame。

因此 Restart Count 使用下限判定，不在等待 2 ms 後要求固定等於 Done Count。

### 4.2 Block/Wave 測試再加入

```txt
spiB_slave.u16BlockStatus
spiB_slave.u16BlockWriteIndex
spiB_slave.u16BlockExpectedLen
spiB_slave.u16BlockChecksum
spiB_slave.u16BlockExpectedChecksum
spiB_slave.u16BlockErrorCode
spiB_slave.u16BlockProgress
spiB_slave.u32BlockRxCount
g_u16SpiBlockRam
g_u16SpiMasterWaveRam
```

### 4.3 問題發生時再加入

```txt
spiA_master.stApp
spiA_master.stDriver
spiB_slave.fsm
spiB_slave.stat
gSpibRxRegFrame
g_u32DebugLastTx
g_u32DebugLastValidResponse
```

## 5. 通用觸發方式

先設定：

```txt
spiA_master.stTest.eCommand
spiA_master.stTest.u16Address
spiA_master.stTest.u16Data
```

最後設定：

```txt
spiA_master.stTest.u16Start = 1
```

Firmware 會自動把 `u16Start` 清為 0。

結果：

```txt
SPI_TEST_STATUS_IDLE    = 0
SPI_TEST_STATUS_RUNNING = 1
SPI_TEST_STATUS_PASSED  = 2
SPI_TEST_STATUS_FAILED  = 3
```

`u32Detail` 在失敗時記錄 Slave 最後收到的非-null request：
高 16-bit 為 address，低 16-bit 為 data。

每次測試的基本 Pass 條件：

```txt
spiA_master.stTest.eStatus == SPI_TEST_STATUS_PASSED
spiA_master.stDiag.eHealth == SPIA_HEALTH_OK
spiB_slave.stDiag.eHealth == SPIB_HEALTH_OK
gSpibRxParseFailCount 差值 == 0
gSpibRxErrorFlags == 0
```

## 6. Test 1：Legacy Register Write

目的：確認既有 2-word register write、Slave DMA 與回覆 checksum。

設定：

```txt
spiA_master.stTest.eCommand   = SPI_MASTER_TEST_CMD_WRITE
spiA_master.stTest.u16Address = 0x0401
spiA_master.stTest.u16Data    = 0x1234
spiA_master.stTest.u16Start   = 1
```

預期：

```txt
spiA_master.stTest.eStatus  = SPI_TEST_STATUS_PASSED
spiA_master.stTest.u16Result = 0x1234
g_u32DebugLastValidResponse = 0x04471234
```

說明：

```txt
byte checksum = 0x12 + 0x34 = 0x0046
response address = 0x0401 + 0x0046 = 0x0447
```

`0x0400` 是 CPU1 Version 特殊位址，固定回覆 `0x8A90`，不可用來驗證
`0x1234` echo write。

Slave DMA Counter 預期差值：

```txt
gSpibRxDmaDoneCount    +2
gSpibRxParseOkCount    +2
gSpibRxParseFailCount  +0
gSpibRxDmaRestartCount 至少 +2
```

`+2` 來自一筆 request frame 與一筆 clock response 用的 null frame。

- [x] PASS
- [ ] FAIL

## 9. Test 4：16-word Block Write

目的：確認 block address、資料順序、checksum、end command 與 background commit。

設定：

```txt
spiA_master.stTest.eCommand   = SPI_MASTER_TEST_CMD_SEQ_WRITE_16
spiA_master.stTest.u16Address = 0
spiA_master.stTest.u16Data    = 0
spiA_master.stTest.u16Start   = 1
```

`g_u16SpiBlockRam[0..15]` 預期：

```txt
1111 2222 3333 4444 5555 6666 7777 8888
9999 AAAA BBBB CCCC DDDD EEEE FFFF 1110
```

完成後預期：

```txt
spiA_master.stTest.eStatus = SPI_TEST_STATUS_PASSED
spiB_slave.u16BlockWriteIndex = 16
spiB_slave.u16BlockExpectedLen = 16
spiB_slave.u16BlockChecksum = 0x1011
spiB_slave.u16BlockExpectedChecksum = 0x1011
spiB_slave.u16BlockErrorCode = 0
spiB_slave.u16BlockProgress = 100
spiB_slave.u16BlockStatus = 3
```

Slave DMA Counter 預期差值：

```txt
Done 至少 +21
Parse OK 差值 == Done 差值
Parse Fail +0
Restart 差值 >= Done 差值
```

原因：

```txt
16 data + 1 checksum + 1 end + 1 null = 19 frames
完成前至少執行 1 次 Status read = 2 frames
```

- [x] PASS
- [ ] FAIL

## 12. Test 7：1000 次 Legacy Frame DMA 壓測

目的：確認 Slave DMA 可以連續完成 1000 筆固定 2-word frame。

測試前記錄 Counter baseline，或重新啟動 CPU。

設定：

```txt
spiA_master.stTest.eCommand   = SPI_MASTER_TEST_CMD_REG_FRAME_1000
spiA_master.stTest.u16Address = 0
spiA_master.stTest.u16Data    = 0
spiA_master.stTest.u16Start   = 1
```

Master 預期：

```txt
spiA_master.stTest.eStatus = SPI_TEST_STATUS_PASSED
spiA_master.stTest.u16Result = 1000
```

Slave Counter 預期差值：

```txt
gSpibRxDmaDoneCount    = 1000
gSpibRxParseOkCount    = 1000
gSpibRxParseFailCount  = 0
gSpibRxDmaRestartCount >= 1000
gSpibRxErrorFlags      = 0
```

實際判定請使用：

```txt
測試後 Counter - 測試前 Counter
```

若在最後一筆後超過 2 ms 才讀值，silence recovery 可能使
`gSpibRxDmaRestartCount` 成為 1001，這不代表多收到一筆 frame。

另外預期最後一筆 data 為 `999`，因此：

```txt
OUTPUT_ON = 1
```

- [ ] PASS
- [ ] FAIL

## 13. Test 8：Packet Write

目的：確認 Packet mode 可透過共同 handler 寫入與 Legacy 相同的 register。

設定：

```txt
spiA_master.stTest.eCommand   = SPI_MASTER_TEST_CMD_PACKET_WRITE
spiA_master.stTest.u16Address = 0x0900
spiA_master.stTest.u16Data    = 0x0001
spiA_master.stTest.u16Start   = 1
```

線上傳送內容：

```txt
Frame 0: 0xA55A, 0x0900
Frame 1: 0x0001, 0x0001
Frame 2: 0xAE5C checksum, 0x0000 padding
Frame 3: 0xFFFF, 0x0000 null
```

預期：

```txt
spiA_master.stTest.eStatus = SPI_TEST_STATUS_PASSED
spiA_master.stTest.u16Result = 4
OUTPUT_ON = 1
spiB_slave.ePacketState = SPIB_PACKET_STATE_IDLE
spiB_slave.u16PacketErrorCode = SPIB_PACKET_ERROR_NONE
spiB_slave.stDiag.stProtocol.stComm.u32RxTotal 差值 = 1
```

Slave DMA Counter 預期差值：

```txt
Done +4, Parse OK +4, Parse Fail +0, Restart 至少 +4
```

重要：Master 的 Packet test 目前只確認四個 frame 已送完，沒有驗證 response。
本測試必須同時確認 `OUTPUT_ON` 與 Slave packet error，才可判定功能通過。

### 13.1 共存確認

Packet test 完成後，再執行 Test 3 Clear：

```txt
Legacy write 0x0900, 0x0000
```

預期：

```txt
OUTPUT_ON = 0
Legacy test PASSED
```

這可確認 Packet mode 結束後 parser 已回到 Legacy mode。
