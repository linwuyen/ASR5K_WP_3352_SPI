可以用兩種方式執行：

- CCS Expressions：操作 `spiA_master.stApp`
- 程式碼：呼叫固定 SPIA wrapper API

執行前確認主迴圈持續呼叫：

```c
runSPIBslave();
runSPIAmaster();
```

**1. 單筆讀取**

CCS Master 設定：

```text
spiA_master.stApp.u16TestAddr = 0x0400
spiA_master.stApp.eTestCmd    = SPI_MASTER_TEST_CMD_READ
spiA_master.stApp.eTestState  = SPI_MASTER_TEST_TRIGGER
```

或程式碼：

```c
readSPIAmaster(C2000_Version_spi_addr, callback);
```

Slave 檢查：

```text
g_u32DebugLastValidResponse = 0x051A8A90
```

Master 成功後：

```text
spiA_master.stApp.u16TestData = 0x8A90
spiA_master.stApp.eTestState  = SPI_MASTER_TEST_DONE
spiA_master.stApp.u16LastResult = 0x8A90
```

---

**2. 單筆寫入**

建議使用真正可寫入的 `0x0900`：

```text
spiA_master.stApp.u16TestAddr = 0x0900
spiA_master.stApp.u16TestData = 0x0001
spiA_master.stApp.eTestCmd    = SPI_MASTER_TEST_CMD_WRITE
spiA_master.stApp.eTestState  = SPI_MASTER_TEST_TRIGGER
```

或程式碼：

```c
writeSPIAmaster(Output_ON_OFF_spi_addr, 1U, callback);
```

Slave 檢查：

```text
OUTPUT_ON       = 1
g_u32DebugLastValidResponse = 0x09010001
```

Master 成功後：

```text
spiA_master.stApp.eTestState  = SPI_MASTER_TEST_DONE
spiA_master.stApp.u16LastResult = 0x0001
```

---

**3. 一般 Block 傳輸**

CCS 內建 16 點順序寫入測試：

```text
spiA_master.stApp.eTestCmd   = SPI_MASTER_TEST_CMD_SEQ_WRITE_16
spiA_master.stApp.eTestState = SPI_MASTER_TEST_TRIGGER
```

Master 會傳送：

```text
0x3000 ～ 0x300F：16 點資料
0x3FFF：Block End
```

程式碼自訂 Block：

```c
writeBlockSPIAmaster(txBuffer, rxBuffer, length, callback);
```

Slave 檢查：

```text
g_u16SpiBlockRam[0] ～ [15]
spiB_slave.u16BlockWriteIndex = 16
spiB_slave.u16BlockExpectedLen = 16
spiB_slave.u16BlockStatus = 3        // READY
spiB_slave.u16BlockReady = 1
spiB_slave.u32BlockRxCount
```

---

**4. 4095 點波形 Block 傳輸**

Ramp 波形：

```text
spiA_master.stApp.eTestCmd   = SPI_MASTER_TEST_CMD_WAVE_4095
spiA_master.stApp.eTestState = SPI_MASTER_TEST_TRIGGER
```

Sine 波形：

```text
spiA_master.stApp.eTestCmd   = SPI_MASTER_TEST_CMD_SINE_4095
spiA_master.stApp.eTestState = SPI_MASTER_TEST_TRIGGER
```

注意：`g_u16SpiMasterWaveRam` 是 Master 端模擬 3352 的送出資料來源，放在 `spia_master_wave_ram -> RAMGS0`，上電後保持 0；只有觸發 Sine 波形測試時才會填入 sine table。Slave 是否真的收到，請看 `g_u16SpiBlockRam[0] ～ [4094]`，它放在 `spib_block_ram -> RAMGS2`。

或程式碼：

```c
writeWaveBlockSPIAmaster(0U, callback); // Ramp
writeWaveBlockSPIAmaster(1U, callback); // Sine
```

Slave 檢查：

```text
g_u16SpiBlockRam[0] ～ [4094]
spiB_slave.u16BlockWriteIndex = 4095
spiB_slave.u16BlockExpectedLen = 4095
spiB_slave.u16BlockStatus = 3
spiB_slave.u16BlockReady = 1
spiB_slave.u16BlockProgress = 100
```

Ramp 預期值：

```text
g_u16SpiBlockRam[0]    = 0x0010
g_u16SpiBlockRam[1]    = 0x0020
g_u16SpiBlockRam[4094] = 0xFFF0
```

---

**5. SPIB RX DMA 2-word register frame 壓測**

這個測試走 raw 2-word frame，不走一般 `writeSPIAmaster()` request/response 包裝，所以不會額外送 `0xFFFF,0x0000` null frame。

建議先在 CCS Watch 將 Slave debug counter 清為 0，或從程式呼叫：

```c
SPIB_RxDmaResetDebugCounters();
```

CCS Master 觸發：

```text
spiA_master.stApp.eTestCmd   = SPI_MASTER_TEST_CMD_REG_FRAME_1000
spiA_master.stApp.eTestState = SPI_MASTER_TEST_TRIGGER
```

Master 會連續送 1000 筆 2-word register frame：

```text
cmd  = Output_ON_OFF_spi_addr // 0x0900 fast path
data = 0, 1, 2, ... 999
```

Slave 預期檢查：

```text
gSpibRxDmaDoneCount    = 1000
gSpibRxParseOkCount    = 1000
gSpibRxParseFailCount  = 0
gSpibRxErrorFlags      = 0
```

錯誤旗標定義：

```text
bit0 = SPIB_RX_ERR_DMA_DONE_TIMEOUT
bit1 = SPIB_RX_ERR_RX_FIFO_OVERFLOW
bit2 = SPIB_RX_ERR_DMA_ERROR
bit3 = SPIB_RX_ERR_FRAME_PARSE_FAIL
```

Master 完成檢查：

```text
spiA_master.stApp.eTestState        = SPI_MASTER_TEST_DONE
spiA_master.stApp.u16LastResult     = 1000
spiA_master.stApp.u16RawFrameIndex  = 1000
spiA_master.stApp.u16RawFrameTarget = 1000
```
